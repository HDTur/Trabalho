1. Para que serve o commit e qual a melhor forma de utilizá-lo? O commit serve para podermos a transação de softwares ou programas para um repositório. E a melhor forma de utilizar ele seria após a conclusão de cada parte do projeto. 2. Qual a diferença entre commit e push? O commit ele é utilizado para registar as alterações no repositório enquanto o push ele faz a atualização dos repositórios locais com os repositórios que estão alocados em outros locais como na web ou na nuvem. 3. O que é tag? Tag é uma forma geralmente para utilizada para colocar a numeração da versão do software ou do programa. Ela nada mais é que uma marcação utilizada pelas ferramentas de versionamento.
